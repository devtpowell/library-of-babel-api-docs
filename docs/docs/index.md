# Overview

"The Library of Babel" or "La Biblioteca de Babel" is a short story written by Argentinian author Jorge Luis Borges.[^1] 

In it, he describes "a universal library, containing books with every possible combination of 410 pages of letters, thus containing every book that ever has been and every book that ever could be written, drowned out by an immense quantity of nonsense."[^2]

The digital version of the Library of Babel attempts to bring this thought experiment to reality, and this real documentation refers to a fictional API which could be used to access the real digital Library.

## A Question of Verisimilitude

We have a question of real and a question of invented presented to us almost immediately. "La Biblioteca de Babel," for instance, is a real (yet fictional) story written about an library invented by Borges. The digital Library of Babel is a real, but incomplete, version of Borges' imagined and invented Biblioteca. 

His fictional Biblioteca is in another sense a "real" place, full of books made of paper and glue, "real" in a way that a digital record is not. But his Biblioteca is not quantifiable and queriable in the way that the real digital Library is. 

This documentation is "real" in that I've written it, published it, and have a Git record of commits, but it is not real in that the digital Library has no Access API. But it could be created easily enough should the Library's developers want to do so. 

And this documentation has a function: it presents that I am capable of writing API documentation, and it presents you, the reader of this index file, with a thought experiment on the supposed differences between reality and invention.

### A Means of Verification

If we were to question the realness of this documentation, we could, for instance, use the Library itself as a means of verification. For instance, the following string from this document

```
this documentation is real in that ive written it, published it, and have a git record of commits, but it is not real in that the digital library has no access api. but it could be created easily enough should the librarys developers want to do so.
```

exists in many books in the library. For instance, it exists in the book titled *ucevgol uxk,lfbqjjafa,b* on page 287.[^3] It is readily available and has already been written. Somewhere else in the library, the entirety of this documentation exists, as well as a simple tutorial on how to create the API which it documents.

## Usage

This API can be used to query books by string, as in the example above, or to search by Hexagon, Wall, Shelf, and finally, Volume. Each Hexagon contains 4 walls; each wall contains 5 shelves, and each shelf contains 32 volumes. Each volume contains 410 pages, each page contains 3200 characters. The characters in and of themselves don't contain anything.
